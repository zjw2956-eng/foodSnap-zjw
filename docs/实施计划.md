# 美食分享社交平台 — 实施计划

## 一、项目目的

一个基于地理位置的美食分享 UGC 社交平台。用户拍照上传午餐/晚餐美食照片，AI 辅助识别菜品并生成精致文案，其他用户以弹幕形式评论互动、点赞，根据综合评分生成每日区域美食榜单，解决"今天吃什么"的日常选择困难。

## 二、核心链路

```
用户拍照上传 → Minio 存储图片 → 异步触发 AI 分析(Python 服务)
    → AI 识别菜品 + 生成文案 + 同款建议 → 帖子发布
    → 进入所在区域实时热榜池(每 30 分钟刷新)
    → 浏览者点赞 + 弹幕评论 + 收藏 + AI 获取同款建议
    → 每日 0 点结算每日金榜(锁定不可变)
    → 用户主页展示上榜成就

双榜机制：
┌────────────┬──────────────┬─────────────────────────┐
│ 实时热榜    │ 每 30 分钟刷新 │ 近 6 小时帖子按热度排序    │
│ 每日金榜    │ 每日 0 点锁定  │ 综合评分(点赞+评论+收藏)   │
└────────────┬──────────────┴─────────────────────────┘
```

## 三、项目功能

### 3.1 用户模块（已有基础）
- 注册/登录（Shiro 认证，已有）
- 个人主页：发帖数、获赞数、上榜次数
- 修改资料：头像、昵称、邮箱

### 3.2 发帖模块
- 上传美食照片（Minio 存储，限制大小和格式）
- 系统自动收集用户定位信息
- 选择/自动归类美食分类（早餐/午餐/晚餐/夜宵/小吃）
- 异步触发 AI 分析：识别菜品名 → 生成精致文案 → 生成同款建议
- 用户可手动修改 AI 生成的文案后发布
- 帖子状态：正常 / AI 分析中 / 违规下架

### 3.3 浏览互动模块
- 首页信息流：按区域推荐帖子（优先展示所在区域）
- 弹幕评论：在美食图片上飘过，可自定义颜色
- 传统评论列表：附带在帖子详情页，可回复
- 点赞：一键点赞/取消，影响热榜排序
- 收藏：收藏喜欢的帖子，可查看收藏列表
- AI 同款建议：浏览者可点击获取"如何在附近吃到同款"

### 3.4 排行榜模块
- 实时热榜：近 6 小时高赞帖子，每 30 分钟刷新，Redis 缓存
- 每日金榜：每日 0 点 Quartz 定时任务结算，分数公式综合计算
- 区域维度：默认用户所在区域，可切换更大区域/全城/全局
- 分类维度：早餐榜/午餐榜/晚餐榜/综合榜
- 历史榜单可回溯

### 3.5 AI 助手模块（Python 独立服务）
- 图片菜品识别 + 美食知识科普（菜品食材/热量小知识，趣味呈现，非健康建议）
- 为发帖者生成精致朋友圈文案
- 为浏览者提供"吃到同款"建议（附近餐厅推荐）
- Java 主服务通过 HTTP REST 调用 Python 服务
- 异步处理，超时降级

## 四、技术栈

### 已有基础
Spring Boot 2.0.5 + Apache Shiro + MyBatis/tk.MyBatis + Redis + Flyway + Thymeleaf + PageHelper + Druid + Swagger

### 新增引入

| 技术 | 用途 | 优先级 |
|------|------|--------|
| **Minio** | 美食图片对象存储，S3 兼容 | 高，发帖核心依赖 |
| **Redisson** | 分布式锁(排行榜结算)、原子计数 | 高，榜单计算需要 |
| **RocketMQ** | 发帖异步化、AI 分析事件驱动 | 高，已接入 |
| **Quartz**(已有) | 每日金榜定时结算、热榜定时刷新 | 已有，直接使用 |
| **Python FastAPI** | AI 分析独立服务 | 中后期，独立部署 |

## 五、数据库表概要

| 表名 | 用途 | 关键字段 |
|------|------|---------|
| `sys_user` | 用户（已有，需补充） | 新增 nickname、avatar_url、location 等字段 |
| `food_post` | 美食帖子 | 用户、图片、分类、定位、AI 文案、状态、统计数 |
| `food_like` | 点赞记录 | 帖子-用户唯一约束，防重复点赞 |
| `food_comment` | 弹幕评论 | 弹幕位置/颜色、传统评论标记 |
| `food_ranking` | 每日榜单 | 日期、类型(实时/金榜)、区域、分类、排名 |
| `food_collect` | 收藏记录 | 帖子-用户唯一约束 |
| `food_ai_analysis` | AI 分析记录 | 识别结果、置信度、建议文案、处理状态 |
| `food_sensitive_word` | 敏感词 | 弹幕/评论过滤黑名单 |

详细建表 SQL 见 `docs/sql/food_snap_schema.sql`。

## 七、注意事项

1. **图片安全**：UGC 图片上传必须限制大小（建议 10MB）、校验格式（仅允许 jpg/png/webp），后续需要内容审核机制
2. **定位隐私**：收集用户位置需在前端明确告知用途，后端对坐标做模糊化处理（保留到小数点后 2 位，约 1km 精度）
3. **AI 异步超时**：调用 Python 服务设 30s 超时，失败时帖子正常发布但标记"AI 分析中"，提供手动重试
4. **金榜幂等**：每日结算任务必须保证幂等（Redisson 分布式锁 + 结算状态标记），防止重复执行
5. **弹幕敏感词**：评论内容需过敏感词过滤（可先用简单的敏感词列表，后续接入第三方审核）
6. **数据库字符集**：统一使用 `utf8mb4`，支持 emoji 表情
7. **点赞并发**：使用 Redis 缓存点赞数，定时同步到数据库，避免高并发直接打 DB
8. **前后端分离考虑**：当前 Thymeleaf 是服务端渲染，后续如需移动端可改造成 REST API
